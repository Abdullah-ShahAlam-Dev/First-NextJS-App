Step 1
Syntax:
git subtree split --prefix="Folder-Name" -b temp-branch
yeh command kya karti hai?
Ye aapke poore project mein se sirf us ek specific folder ka data aur uske purane commits filter karti hai, aur usay utha kar ek nayi alag duniya (temp-branch) mein rakh deti hai.

git subtree split chalate ho, toh Git ek Filter ki tarah kaam karta hai. Wo tumhari SMIT ki badi history mein se sirf aur sirf un commits ko dhundta hai jo Github/advance-github wale folder ke andar hue thay.
Wo unhi specific commits ko nikal kar ek "Clean History" banata hai. Isliye jab tum push karte ho, toh remote repo bilkul saaf hoti hai aur usme SMIT ke doosre kisi project ka koi commit ya history nahi jati.


creating temprory bracnh for force push
git subtree split --prefix="Github/advance-github" -b temp-branch
git subtree split --prefix="PortableGit" -b temp-git


Step 2
Syntax:
git push -f <remote-url-ya-name> <local-branch-name>:<remote-branch-name>
so, for Instance:
git push -f https://github.com/Abdullah-ShahAlam-Dev/advance-github.git temp-branch:feature/navbar
git push -f https://github.com/Abdullah-ShahAlam-Dev/portable-git.git temp-git:main


Step 3
Apna data upload karne ke baad, aapko ab us local temp-branch ki zaroorat nahi hai. Wo sirf ek "delivery boy" ka kaam kar rahi thi. Isliye aapne git branch -D temp-branch karke usay delete kar diya taake aapka local system clean rahay aur extra branches jama na hon.
git branch -D temp-branch
git branch -D temp-git

COMPLETE FLOW:
Poora Workflow Ek Saath Kaise Kaam Karta Hai?
Jab aap yeh commands chalate hain, toh background mein ye logic chal raha hota hai:

1.Extract & Isolate (Subtree Split): Aapne kaha ke sirf Github/advance-github folder ko pakro aur uska sara data temp-branch naam ki branch mein daal do. (Ab aapka main project safe hai, aur ek alag copy branch mein ban gayi hai).

2.Upload (Force Push): Phir aapne us temp-branch ko pakra aur us remote URL par -f ke sath push kar diya. Isne remote repository mein sirf us specific folder ka data daal diya.

3.Clean up (Branch Delete): Apna data upload karne ke baad, aapko ab us local temp-branch ki zaroorat nahi hai. Wo sirf ek "delivery boy" ka kaam kar rahi thi. Isliye aapne git branch -D temp-branch karke usay delete kar diya taake aapka local system clean rahay aur extra branches jama na hon.
Summary:
Short words mein: Yeh ek automated tareeqa hai apne bade project mein se hissay (folders) nikal kar unhein unki apni apni alag repositories mein bhejne ka, aur apna local setup saaf rakhne ka.
